## Overview

A global wallet allows users to deposit tokens on any supported chain and spend them seamlessly on any other chain using a single account address. This creates a unified balance across all chains while maintaining the same address everywhere.

**Key benefits:**
- Single account address across all chains
- Deposit on cheap chains, spend on expensive chains
- Automatic liquidity aggregation
- Simplified user experience

## Create an Account

```tsx
import { createRhinestoneAccount } from '@rhinestone/sdk'
import { base, arbitrum } from 'viem/chains'
import { generatePrivateKey, privateKeyToAccount } from 'viem/accounts'

const privateKey = generatePrivateKey()
const owner = privateKeyToAccount(privateKey)

const account = await createRhinestoneAccount({
  owners: {
    type: 'ecdsa',
    accounts: [owner],
  },
  rhinestoneApiKey: process.env.RHINESTONE_API_KEY,
})

const address = account.getAddress()
console.log('Global wallet address:', address)
// You can receive funds to this address on any supported chain.
```

## Deposits

- Users can send tokens directly to the account address on any supported chain.
- Deposits are available for cross-chain spending; no extra setup is required in the SDK.

## Spend Across Chains

Send tokens on a destination chain, sourcing funds from a source chain:

```tsx
import { encodeFunctionData, erc20Abi, parseUnits } from 'viem'
import { arbitrum, base } from 'viem/chains'

const recipient = '0x0000000000000000000000000000000000000001'
const baseUSDC = '0x0000000000000000000000000000000000000002' // USDC on Base

const tx = await account.sendTransaction({
  sourceChains: [arbitrum],
  targetChain: base,
  calls: [
    {
      to: baseUSDC,
      data: encodeFunctionData({
        abi: erc20Abi,
        functionName: 'transfer',
        args: [recipient, parseUnits('100', 6)], // 100 USDC
      }),
    },
  ],
  tokenRequests: [
    { address: baseUSDC, amount: parseUnits('100', 6) },
  ],
})

const result = await account.waitForExecution(tx)
console.log('Status:', result.status)
```
