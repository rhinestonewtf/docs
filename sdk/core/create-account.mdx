Let's create a smart account with a single owner.

## Choosing Your Signer

Before creating your account, you need to decide how users will sign transactions:

**Option 1: Generate a signer client** (like below) - Use this for testing or when you want full control over key generation.

**Option 2: External wallet integration** - If you're building a dApp that connects to existing wallets (MetaMask, Coinbase, etc.), you can create signers from their providers and pass them around.

**Option 3: Embedded signers** - For the best UX in production apps, use embedded wallet providers like [Privy](./signers/privy), [Dynamic](./signers/dynamic), or [Turnkey](./signers/turnkey) that handle user onboarding and key management.

## Basic Account Creation

```ts
import { createRhinestoneAccount } from '@rhinestone/sdk'
import { generatePrivateKey, privateKeyToAccount } from 'viem/accounts'
import { baseSepolia, arbitrumSepolia } from 'viem/chains'

const rhinestoneApiKey = process.env.RHINESTONE_API_KEY

// You can use an existing PK here
const privateKey = generatePrivateKey()
console.log(`Owner private key: ${privateKey}`)
const account = privateKeyToAccount(privateKey)

const rhinestoneAccount = await createRhinestoneAccount({
  owners: {
    type: 'ecdsa',
    accounts: [account],
  },
  rhinestoneApiKey,
})
const address = await rhinestoneAccount.getAddress()
console.log(`Smart account address: ${address}`)
```

Once you fund the account, you can start making transactions:

```ts
import { getTokenAddress } from '@rhinestone/sdk/orchestrator'
import { baseSepolia } from 'viem/chains'

const chain = baseSepolia
const usdcTarget = getTokenAddress('USDC', chain)
const usdcAmount = 1n
const receiver = '0xd8da6bf26964af9d7eed9e03e53415d37aa96045'

const transaction = await rhinestoneAccount.sendTransaction({
  chain,
  calls: [
    {
      to: usdcTarget,
      data: encodeFunctionData({
        abi: erc20Abi,
        functionName: 'transfer',
        args: [receiver, wethAmount],
      }),
    },
  ],
})
console.log('Transaction', transaction)
```

## Account Customization

### Smart Account Providers

Rhinestone works with any ERC-7579 compatible smart account. You can customize which account implementation to use:

```tsx
// Use different account providers
import { createRhinestoneAccount } from '@rhinestone/sdk'

// Nexus account (default)
const nexusAccount = await createRhinestoneAccount({
  owners: {
    type: 'ecdsa',
    accounts: [account],
  },
  accountProvider: 'nexus', // Default
})

// Safe account
const safeAccount = await createRhinestoneAccount({
  owners: {
    type: 'ecdsa',
    accounts: [account],
  },
  accountProvider: 'safe',
})

// Kernel account
const kernelAccount = await createRhinestoneAccount({
  owners: {
    type: 'ecdsa',
    accounts: [account],
  },
  accountProvider: 'kernel',
})
```

Learn more about [account providers](../customize/smart-account-providers/nexus) and their specific features.

### Initial Modules

Install modules during account creation:

```tsx
import { getWebAuthnValidator, getAutoSavingsExecutor } from '@rhinestone/module-sdk'

const customizedAccount = await createRhinestoneAccount({
  owners: {
    type: 'ecdsa',
    accounts: [account],
  },
  initialModules: [
    getWebAuthnValidator({ pubKeyX: '0x...', pubKeyY: '0x...' }),
    getAutoSavingsExecutor({ savingsRate: 500 }), // 5% auto-save
  ],
})
```

### Multi-Owner Setup

Create accounts with multiple owners:

```tsx
const multiOwnerAccount = await createRhinestoneAccount({
  owners: {
    type: 'multisig',
    accounts: [owner1, owner2, owner3],
    threshold: 2, // 2-of-3 signatures required
  },
})
```
