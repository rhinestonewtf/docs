Smart accounts in Rhinestone require at least one **signer** (owner) to authorize transactions. The type of signer you choose depends on your application's needs and user experience requirements.

## Signer Types Overview

Before creating your account, you need to decide how users will sign transactions:

### Passkeys
Biometric authentication using WebAuthn for password-free, secure signing. Perfect for consumer applications prioritizing security and UX.

- **Best for**: Consumer apps, high-security applications
- **Guide**: [Passkeys Integration](./passkeys)

### Embedded Wallets  
Third-party providers handle key management and user onboarding. Users don't need existing wallets.

- **Best for**: New-to-crypto users, seamless onboarding
- **Providers**: [Dynamic](./signers/dynamic), [Privy](./signers/privy), [Magic](./signers/magic), [Turnkey](./signers/turnkey)

### External Wallets
Connect to existing wallets like MetaMask, Coinbase Wallet, or any WalletConnect-compatible wallet.

- **Best for**: Existing crypto users, dApps, DeFi applications  
- **Guide**: [External Wallet Integration](./signers/external-wallet)

### Generated Keys
Programmatically generated private keys for testing or specialized use cases.

- **Best for**: Testing, backend services, specialized applications
- **Usage**: Shown in examples below

## Quick Start: Generated Key Account

Here's the fastest way to create a smart account for testing or development:

```ts
import { createRhinestoneAccount } from '@rhinestone/sdk'
import { generatePrivateKey, privateKeyToAccount } from 'viem/accounts'
import { baseSepolia, arbitrumSepolia } from 'viem/chains'

const rhinestoneApiKey = process.env.RHINESTONE_API_KEY

// You can use an existing PK here
const privateKey = generatePrivateKey()
console.log(`Owner private key: ${privateKey}`)
const account = privateKeyToAccount(privateKey)

const rhinestoneAccount = await createRhinestoneAccount({
  owners: {
    type: 'ecdsa',
    accounts: [account],
  },
  rhinestoneApiKey,
})
const address = await rhinestoneAccount.getAddress()
console.log(`Smart account address: ${address}`)
```

<Warning>
**Important**: Save the private key securely. You'll need it to sign transactions. In production, use embedded wallets or external wallet connections instead of generated keys.
</Warning>


## Making Your First Transaction

Once your account is created and funded, you can send transactions:

```ts
import { getTokenAddress } from '@rhinestone/sdk/orchestrator'
import { baseSepolia } from 'viem/chains'

const chain = baseSepolia
const usdcTarget = getTokenAddress('USDC', chain)
const usdcAmount = 1n
const receiver = '0xd8da6bf26964af9d7eed9e03e53415d37aa96045'

const transaction = await rhinestoneAccount.sendTransaction({
  chain,
  calls: [
```

## Embedded Wallet Integration

For production applications, choose the appropriate signer type:

- **New users**: Use embedded wallets like [Dynamic](./signers/dynamic) or [Privy](./signers/privy)
- **Existing crypto users**: Support [external wallets](./signers/external-wallet) 
- **High security**: Implement [passkeys](./passkeys) for biometric authentication
- **Enterprise**: Use [Turnkey](./signers/turnkey) for robust key management

## Advanced Configuration

### Account Providers

Rhinestone supports multiple ERC-7579 compatible smart account implementations:

```ts
const account = await createRhinestoneAccount({
  owners: { type: 'ecdsa', accounts: [signer] },
  account: {
    type: 'nexus', // 'safe', 'kernel', 'startale', 'prime'
  },
  rhinestoneApiKey,
})
```

**Learn more**: [Smart Account Providers](../customize/smart-account-providers/nexus)

### Multi-Signature Accounts

Create accounts requiring multiple signatures:

```ts
const multiSigAccount = await createRhinestoneAccount({
  owners: {
    type: 'ecdsa', // ecdsa, passkey, multi-factor
    accounts: [owner1, owner2, owner3],
    threshold: 2, // 2-of-3 signatures required
  },
  rhinestoneApiKey,
})
```

**Learn more**: [Multisig Setup](./multisig)
