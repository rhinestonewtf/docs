---
title: "Build your first smart wallet"
description: "Create a smart account owned by an external wallet and send your first transaction."
---

Build a browser app with wallet connection using an external wallet (MetaMask or any WalletConnect-compatible wallet) as the signer. No third-party provider required — just your wallet and the Rhinestone SDK. If you just want to validate the SDK works without a browser setup, start with the [Quickstart](../quickstart) instead.

<Info>Want to use a different signer? See [Signer types](../core/setup-signer) for passkeys, embedded wallets, and more.</Info>

## Prerequisites

- Node.js 22+
- An API key from the [Rhinestone dashboard](https://dashboard.rhinestone.dev)
- MetaMask or another browser wallet installed

## Steps

<Steps>
  <Step title="Install dependencies">

    ```bash
    npm install viem @rhinestone/sdk @reown/appkit @reown/appkit-adapter-wagmi wagmi
    ```

  </Step>
  <Step title="Connect the wallet">

    Use Reown AppKit to connect the user's external wallet and get a Viem wallet client:

    ```ts
    import { createAppKit } from '@reown/appkit'
    import { WagmiAdapter } from '@reown/appkit-adapter-wagmi'
    import { arbitrum, base } from '@reown/appkit/networks'
    import { useWalletClient } from 'wagmi'

    const projectId = process.env.REOWN_PROJECT_ID as string

    const wagmiAdapter = new WagmiAdapter({
      networks: [arbitrum, base],
      projectId,
    })

    createAppKit({
      adapters: [wagmiAdapter],
      networks: [arbitrum, base],
      projectId,
      metadata: {
        name: 'My App',
        description: 'My Rhinestone app',
        url: 'https://myapp.com',
        icons: ['https://myapp.com/icon.png'],
      },
    })
    ```

    Render the connect button in your UI:

    ```tsx
    export function ConnectButton() {
      return <w3m-button />
    }
    ```

  </Step>
  <Step title="Create the smart account">

    Once the wallet is connected, wrap the Viem wallet client and create the Rhinestone account:

    ```ts
    import { RhinestoneSDK, walletClientToAccount } from '@rhinestone/sdk'
    import { useWalletClient } from 'wagmi'

    const { data: walletClient } = useWalletClient()

    const rhinestone = new RhinestoneSDK({
      apiKey: process.env.RHINESTONE_API_KEY as string,
    })

    const signer = walletClientToAccount(walletClient)

    const account = await rhinestone.createAccount({
      owners: {
        type: 'ecdsa',
        accounts: [signer],
      },
    })

    const accountAddress = account.getAddress()
    ```

    The account address is deterministic — it's derived from the owner configuration and doesn't require a deployment transaction.

  </Step>
  <Step title="Send a transaction">

    Send a crosschain USDC transfer from Arbitrum to Base:

    ```ts
    import { arbitrum, base } from 'viem/chains'
    import { encodeFunctionData, erc20Abi, parseUnits } from 'viem'

    const recipientAddress = '0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045'

    const result = await account.sendTransaction({
      sourceChains: [arbitrum],
      targetChain: base,
      calls: [
        {
          to: '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913', // USDC on Base
          data: encodeFunctionData({
            abi: erc20Abi,
            functionName: 'transfer',
            args: [recipientAddress, parseUnits('10', 6)],
          }),
        },
      ],
      tokenRequests: [
        {
          address: '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913',
          amount: parseUnits('10', 6),
        },
      ],
    })
    ```

    MetaMask will prompt the user to sign. The Rhinestone orchestrator handles routing and settlement — no bridging required.

  </Step>
  <Step title="Wait for execution">

    ```ts
    const status = await account.waitForExecution(result)
    ```

  </Step>
</Steps>
